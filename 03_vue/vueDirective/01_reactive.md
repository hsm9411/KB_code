# Vue 반응형 시스템 (Reactivity)

Vue의 가장 핵심 개념. 이것만 이해하면 나머지는 응용이다.

---

## 반응형이란?

**데이터가 바뀌면 화면이 자동으로 바뀐다.**

일반 JS는 이렇게 해야 한다:
```js
// 일반 JS: 직접 DOM을 건드려야 함
document.getElementById('name').innerText = '홍길동'  // 수동
```

Vue는 이렇게 된다:
```js
// Vue: data만 바꾸면 화면이 알아서 바뀜
this.name = '홍길동'  // 끝. 화면 자동 업데이트
```

---

## 동작 원리 (도식)

```
  ┌──────────────────────────────────────┐
  │           Vue 반응형 시스템            │
  │                                      │
  │   data() {                           │
  │     return { name: '홍길동' }         │
  │   }                                  │
  │       │                              │
  │       ▼                              │
  │   Vue가 Proxy로 감싸서 감시 등록       │
  │       │                              │
  │       ▼                              │
  │   name이 사용된 곳을 추적 (의존성 수집) │
  │       │                              │
  │       ▼                              │
  │   name 변경 감지 → 연결된 DOM만 업데이트 │
  └──────────────────────────────────────┘
```

```
this.name = '김영희'   (setter 호출)
        │
        ▼
    Vue: "name 바뀜"
        │
        ▼
    name을 쓰는 {{ name }}, v-text 등 찾기
        │
        ▼
    해당 DOM만 다시 렌더링 (전체 페이지 X)
```

---

## 반응형이 되는 것 / 안 되는 것

```
✅ 반응형 O
─────────────────────────
data() { return { ... } }  ← 여기 선언된 것
this.name = '새값'          ← 기존 속성 변경
this.list.push('항목')      ← 배열 메서드

❌ 반응형 X
─────────────────────────
const x = '값'             ← data 밖에서 선언
this.newProp = '값'        ← data에 없던 새 속성 추가 (Vue 2)
```

> Vue 3에서는 Proxy 기반이라 새 속성 추가도 감지되지만,
> data()에 미리 선언하는 습관이 좋다.
